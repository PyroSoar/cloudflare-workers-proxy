Cloudflare Workers Proxy 使用说明

[ServerDomain]https://proxy1.lzc2002.top。

本代理服务器提供两种调用方式：/api/和/fetch/。它们的用途和限制不同，适合不同场景。

备用地址：Proxy2 服务器

/api/ 接口

/api/用于客户端调用 API，支持多种请求方法（GET、POST、PUT、PATCH、DELETE）。

  • 请求方法：保持原始方法。
  • 请求头：会透传客户端请求头。
  • 请求体：支持 POST/PUT 等方法的请求体。
  • 响应处理：根据Content-Type返回 JSON 或文本。
  • 限制：只允许以下 Content-Type：
    • text/*
    • application/json
    • application/xml
    • application/x-www-form-urlencoded
    • multipart/form-data
    • application/octet-stream
    • application/x-ndjson
    • application/vnd.api+json
  • 大小检查：先通过 HEAD 请求判断文件大小，超过 1MB 会直接拒绝。
示例：
GET [ServerDomain]/api/https://api.example.com/data
POST [ServerDomain]/api/https://api.example.com/submit

/fetch/ 接口

/fetch/用于浏览器直接访问资源，始终使用 GET 方法。

  • 请求方法：统一为 GET。
  • 请求头：不透传客户端请求头。
  • 请求体：不支持。
  • 响应处理：返回原始二进制数据（如图片、文件）。
  • 大小检查：同样通过 HEAD 请求判断文件大小，超过 1MB 会直接拒绝。
  • 类型检查：不限制 Content-Type。
示例：
GET [ServerDomain]/fetch/https://example.com/image.png
GET [ServerDomain]/fetch/https://example.com/file.zip

使用场景

1. 代理地区封锁的网址

例如：https://api.openai.com/v1/chat/completions在某些地区无法直接访问。

通过/api/代理方式，可以绕过封锁：

curl -X POST \
[ServerDomain]/api/https://api.openai.com/v1/chat/completions \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: application/json" \
-d '{"model":"gpt-4","messages":[{"role":"user","content":"Hello"}]}'

2. 代理仅限本域名使用的限制（伪造 Referer）

有些资源只允许在特定域名下访问，通常会检查Referer或Origin。

通过/fetch/代理方式，可以伪装成本域名访问：

<img src="[ServerDomain]/fetch/https://restricted-site.com/image.png" />
